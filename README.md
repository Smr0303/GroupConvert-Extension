# GroupConvert — Chrome Extension

A Chrome extension that captures lead data (names, emails, screening question answers) from Facebook Group membership requests and pushes it to Google Sheets.

Built with [Plasmo](https://www.plasmo.com/) — React + TypeScript, Manifest V3.

---

## How It Works

```
Facebook Group Member Requests Page (DOM)
        |
        v
Chrome Extension (content script scrapes DOM)
        |
        v  POST /api/leads
Backend API (Express + Supabase)
        |
        +---> Saves leads to Supabase (Postgres)
        +---> Pushes rows to Google Sheets API v4
                    |
                    v
            User's Google Sheet
```

1. Navigate to any Facebook Group's **Member Requests** page
2. The extension automatically detects and parses pending requests
3. A floating overlay ("G" button) appears with captured lead count
4. Click **Capture & Push** to send leads to your Google Sheet
5. Optionally toggle **Auto-Approve** to approve requests with randomized delays

---

## Features

- **Side Panel UI** — persistent panel that stays open across tab switches
- **Automatic DOM Scraping** — MutationObserver watches for new member request cards (including infinite scroll)
- **Tiered Selectors** — uses ARIA labels, roles, and structural patterns (not fragile CSS classes)
- **Google Sheets Integration** — one-click OAuth, automatic row appending
- **Auto-Approve** — sequential approval with random 1-3s delays to avoid rate limiting (capped at 50/session)
- **State Persistence** — all settings saved to `chrome.storage.local`

## Setup

### Prerequisites

- Node.js 18+
- pnpm (`npm install -g pnpm`)
- [GroupConvert Service](https://github.com/Smr0303/GroupConvert-Service) backend running

### Install & Run

```bash
# Install dependencies
pnpm install

# Start development server (with HMR)
pnpm dev

# Production build
pnpm build

# Production build + zip for distribution
pnpm build --zip
```

### Environment Variables

Create a `.env` file:

```env
PLASMO_PUBLIC_BACKEND_URL=http://localhost:3000
```

For production, point to the deployed backend URL.

### Load in Chrome

1. Run `pnpm dev` (or `pnpm build` for production)
2. Open `chrome://extensions/`
3. Enable **Developer mode**
4. Click **Load unpacked**
5. Select `build/chrome-mv3-dev/` (dev) or `build/chrome-mv3-prod/` (prod)

## Usage

1. **Click the extension icon** — opens the side panel
2. **Enter any license key** — creates/authenticates your account
3. **Connect Google Sheets** — OAuth flow opens in a new tab
4. **Enter your Google Sheet ID** — paste the sheet URL or ID
5. **Navigate to a Facebook Group's Member Requests page**
6. The extension auto-scrapes visible requests
7. Use the **floating "G" button** or side panel to push leads to your sheet

## DOM Scraping Strategy

Facebook uses obfuscated CSS class names that change frequently. This extension uses stable selectors:

| Priority | Strategy | Example |
|----------|----------|---------|
| 1 | ARIA labels | `[aria-label^="Approve "]` |
| 2 | Roles | `[role="button"]`, `[role="link"]` |
| 3 | Structural | `a[href*="/user/"]`, `ul > li` |
| 4 | Walk-up | Find approve button → traverse up to card container |
